# Умный склад

Это базовая структура склада, где клиент сам создает товары без хардкода.

1. Установить и авторизовать `Supabase CLI`
2. Создать проект в Supabase
3. Вставить ключи в `.env`
4. Применить миграцию

## Web админка

В проекте уже есть `Next.js + Supabase` админка для склада.

Запуск:

1. `npm install`
2. `npm run dev`
3. открыть `http://localhost:3000`

Что умеет:

- добавлять материалы
- добавлять профиль, заглушку и фиксатор
- менять цену
- менять единицу измерения
- менять остаток
- скрывать или активировать позицию

## Как работает логика

Склад делится на 4 категории:

1. Материалы
2. Профиль
3. Заглушка
4. Фиксатор

### 1. Материалы

Клиент создает товар по полям:

- категория: `material`
- коллекция: `Лиссабон`, `София`, `Кармен`, `Памела`, `Black out`
- модель: `01`, `02`, `03` и т.д.
- цена
- единица: `м2` или `метр`
- остаток

Пример:

- Лиссабон 01 - 800 c
- София 01 - 1100 c
- Кармен 01 - 1100 c
- Памела 01 - 1000 c
- Black out 01 - 1300 c

### 2. Профиль

Клиент создает товар по полям:

- категория: `profile`
- название: `Профиль`
- цвет: `антрацит`, `белый`, `коричневый`, `серый`
- цена
- единица: `метр`
- остаток

### 3. Заглушка

Клиент создает товар по полям:

- категория: `cap`
- название: `Заглушка`
- цвет: `антрацит`, `белый`, `коричневый`, `серый`
- цена
- единица: `шт` или `пачка`
- остаток

### 4. Фиксатор

Клиент создает товар по полям:

- категория: `fixer`
- название: `Фиксатор`
- цвет: `антрацит`, `белый`, `коричневый`, `серый`
- цена
- единица: `шт` или `пачка`
- остаток

## Главное

Клиент может сам:

- добавлять новые коллекции
- добавлять новые модели
- выбирать цвет
- менять цену
- выбирать единицу измерения
- вести остаток

Система не зависит от жестко заданного списка товаров.
